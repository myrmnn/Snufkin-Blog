October 16, 2019 Liquid Frozen Jello

Layout and positioning chapter

  • A liquid layout is one in which the content of the page expands to fit the page when you expand the browser window.
  • A frozen layout is one in which the width of the content is fixed, and it doesn't expand or shrink with the browser window. This has the advantage of providing you more control over you design, but at the cost of not using the browser width as efficiently.
  • A jello layout is one in which the content width is fixed, but the margins expand and shrink with the browser window. A jello layout usually places the content in the center of the page. This has the same advantages as the frozen layout, but is often more attractive.

Chapter 9 Bullet Points

  • CSS uses a box model to control how elements are used.
  • Boxes consist of the content area and optional padding, border, and margin.
  • The content area contains the content of the element. The padding is used to create visual space around the element. The border surrounds the padding and content and provides a way to visually separate the content. The margin surrounds the border, padding, and content, and allows space to be added between the element and other elements. These are all optional.
  • Use the background-position and background-repeat properties to set the background image.
  • Use the class attribute to style elements together as a group, and use the id attribute to give an element a unique name. You can also use the id attribute to provide a unique style for an element. There should only be one element in a page with a given id. Select elements using their id selector, #.
  • An element can only have one id but it can belong to many classes.
  • You can have more than one stylesheet in your HTML, if two stylesheets have conflicting property definitions , the style sheet that is last in the HTML file will receive preference.
  • You can target devices by using media queries in you link element or the @media rule in your CSS.

October 4, 2019 - Starting my journey :)

This whole learning to code thing happened very organically. I have been seeing a therapist for a few months with the intention of trying to find a clear path forward. I have never really known what I wanted to be when I grew up, I have tried many things and failed miserabley. Seeking help in this area seemed like a productive thing to do and one of the things we talked about was the subject of past hobbies. Out of nowhere the memory of playing around with coding popped into my mind. I had forgotten all about it and in fact did not even realize at the time that that's what I was doing. Really, I was just playing around with HTML blindly, I didn't even know it was called HTML. I would tweak the look of pages on geocities for no reason what-so-ever and ultimately I think their purpose was to serve as a memory for me to look back on now. I think at the time I thought I was going to create websites for my different interests, perhaps an online journal. The point of it though was that it was something I spontaneously did and enjoyed, why not pick it back up?

I went home after uncovering that memory and thought about it to myself. I wondered if I could learn to do it again. I would have to start from scratch because I remembered nothing. A few days into this memory my husband (without any kind of prompting from me) told me that he had been listening to podcasts about studying web developement and that he wanted to start learning online using a website called freeCodeCamp.org.
